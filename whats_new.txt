Version 3.3.0:
- Improved: Made all version strings dynamic in the code, eliminating hardcoded version numbers in request headers and fallbacks.
- Improved: Cleaned up the client source code by removing all internal inline comments.
- Improved: Updated README documentation to include official release download links and detailed contribution guidelines.

Version 3.2.0:
- Improved: Re-implemented geocoding search to use a highly resilient chain starting with Komoot's Photon API (completely keyless, case-insensitive, fast, and structured) and falling back to OpenWeather Geocoding and Overpass QL.
- Improved: Added local JSON caching of search results in the user's NVDA AppData directory to prevent API rate limits and speed up location queries.
- Improved: Added a global keyless AQI fallback using Open-Meteo Air Quality API so Pirate Weather and OpenWeather users always get air quality announcements.
- Improved: Checking the "Automatically detect my location" checkbox now immediately triggers auto-detection in the background and saves/announces the detected location.
- Improved: Enhanced search dialog accessibility (initial focus sets to search box, OK button set as default to support Enter key selection).
- Improved: Cleaned up settings UI by removing the manual Use Current Location button and preventing empty country parentheses from displaying.
- Change: The "Air quality when available" checkbox is now unchecked by default.

Version 3.1.0:
- Improved: Replaced legacy geocoding services (geocode.maps.co, OpenWeather Geocoding, Nominatim) with Overpass API using HTTP POST.
- Improved: Added multi-mirror failover support for Overpass API mirrors to handle server load and timeouts seamlessly.
- Improved: Optimized location searches with fast exact-match queries, failing over to case-insensitive regex searches only when needed.
- Improved: Display the OSM object type (e.g. [Node], [Way], [Relation]) next to location search results in settings.
- Improved: Added automatic Dew Point calculation using the Magnus-Tetens formula in OpenWeather 2.5 fallback mode.
- Improved: Added fallback UV Index API queries using the free /2.5/uvi OpenWeather endpoint.
- Improved: Smart warning suppression under "All Providers" to only issue provider-unavailable warnings if both API keys are configured and one fails.
- Fixed: Unchecked checkboxes are strictly respected and corresponding weather details are completely omitted from reports.

Version 2.0.8 — Final Release:
- Fixed: Update notification now shows exactly once per NVDA session when a newer version exists.
- Fixed: Pressing No on the update dialog suppresses the prompt for the rest of the session only (resets on next NVDA restart so you'll be notified again if still not updated).
- Fixed: After successfully installing an update, the prompt is permanently suppressed for that version — no more repeated prompts after updating.
- Fixed: Version comparison correctly handles all version formats (e.g. 2.0.05 == 2.0.5).
- Fixed: The startup update check now has a session guard — it can never fire more than once per NVDA session.
- Fixed: Update server errors (rate limits, timeouts, invalid manifests) are handled gracefully with specific error messages.
- Improved: Update dialog message now reads: "A new version of Weather Checker is available. Would you like to update now?"
- Improved: geocode.maps.co API key integrated for reliable location search (no more rate-limited errors).

Version 2.0.7:
- Improved: Location search now uses geocode.maps.co as a dedicated keyed geocoding service, eliminating rate-limited errors.


- Fixed: Update notification no longer repeats every startup after pressing No — the skipped version is remembered.
- Fixed: Auto-detect location now tries three IP geolocation services in sequence, so a rate-limited service no longer causes a "Geocoding service rate limited" error.
- Fixed: Version comparison now handles leading-zero version numbers (e.g. 2.0.05 vs 2.0.5) correctly, preventing false update alerts.
